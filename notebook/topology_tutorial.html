Topology tutorial#

In the movie ‚Äúkill Bill‚Äù, all the brutal action happens after the main character wake up after the coma and managed to ‚Äúwiggle your big toe‚Äù. In a similar way, all the functionality of seaduck depends on the Topology class.

As the name suggest, this objects describe how the grids are connected in the horizontal dimensions. This is a simple compact model with high performance. It is also designed with expandability in mind.

For any grid configuration not yet supported by this package, only one function must be added to the code, and, with no (or at least very little) extra effort, this module and therefore the entire package should work for the new configuration.

The magic function is simply: given a center grid point and its index, if you one want to move one grid to the up/down/left/right what would the new index be.

And that‚Äôs it.

Let‚Äôs start with the most fundamental method of the object. Simply finding the neighboring grid

help(tp.ind_tend)

Here,0-3 represent up/down/left/right respectively.

This is all very nice, it is definitely a pain to navigate grids. But how can I apply this to actual applications?

Calculate surface vorticity#

This example is going to use Topology to compute vorticity on ECCO. As this notebook is written, packages like xgcm does not support vorticity calculation on grids with complex topology. I don‚Äôt think I need to tell you it is a pain to write it out yourself.

Without further ado, let‚Äôs begin by defining the points of interest. In this example, it is every single corner points on the dataset.

face, iy, ix = np.indices((13, 90, 90))
face = face.ravel()
iy = iy.ravel()
ix = ix.ravel()

ind = (face, iy, ix)

We can also directly read the grid coords here.

lon = tub.XG[ind]
lat = tub.YG[ind]

dx = tub.dX[ind]
dy = tub.dY[ind]
# Idealy, dxV and dyU should be used for accuracy.

If you are not familiar with LLC grids, here is what it looks like. Different color represent different ‚Äúfaces‚Äù which are not necessarily parallel to each other.

Each corner points are connected to 4 staggered velocity points, for MITgcm indexing, the upper and right one has the same index as the corner, so we could just read in that.

Show code cell source Hide code cell source 
u = np.array(ds.UVELMASS)
v = np.array(ds.VVELMASS)

U_up = u[ind]
V_right = v[ind]

We can use the ind_tend_vec function to figure out the other two indexes for the velocity points.

The next line is going to have a warning, because some of the points we defined on continental Antarctica does not have neighboring points. Since, they are on land, it does not matter as far as we are concerned.

ind_left = tp.ind_tend_vec(ind, 2 * np.ones_like(face), cuvwg="V")
ind_down = tp.ind_tend_vec(ind, 1 * np.ones_like(face), cuvwg="U")

Let‚Äôs just read the velocities at the other points.

U_down = u[tuple(ind_down)]
V_left = v[tuple(ind_left)]

It is actually not as simple as that. Grid are not always parallel to each other. For example, sometimes U becomes V, and V becomes -U.

Well, the Topology object takes care of that as well with little additional effort.

V_down = v[tuple(ind_down)]
U_left = u[tuple(ind_left)]

fc1 = ind[0]
fc2 = ind_left[0].astype(int)
fc3 = ind_down[0].astype(int)

where = np.where(fc1 != fc3)[0]
for i in where:
    faces = np.array([fc1[i], fc3[i]])
    UfromUvel, UfromVvel, _, _ = tp.get_uv_mask_from_face(faces)
    U_down[i] = UfromVvel[1] * V_down[i] + UfromUvel[1] * U_down[i]

where = np.where(fc1 != fc2)[0]
for i in where:
    faces = np.array([fc1[i], fc2[i]])
    _, _, VfromUvel, VfromVvel = tp.get_uv_mask_from_face(faces)
    V_left[i] = VfromVvel[1] * V_left[i] + VfromUvel[1] * U_left[i]

Ha, we got everything we need. Let‚Äôs calculate the velocity and plot it.

vor = (V_right - V_left) / dx + (U_down - U_up) / dy
